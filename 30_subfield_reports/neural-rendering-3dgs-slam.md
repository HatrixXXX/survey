## 研究背景

这个领域不是从单篇 3DGS 论文突然出现的。硬件证据先在 SLAM/VIO 和视觉重建流水线里出现，例如 FPL 2016 的 `Semi-dense SLAM on an FPGA SoC`、HOTCHIPS 2018 的 `Navion...`，以及 FPT 2022 的 `FSLAM...`。<a id="FIELD-NEURAL-RENDERING-001-E001"></a>[FIELD-NEURAL-RENDERING-001-E001](#FIELD-NEURAL-RENDERING-001-E001) <a id="FIELD-NEURAL-RENDERING-001-E002"></a>[FIELD-NEURAL-RENDERING-001-E002](#FIELD-NEURAL-RENDERING-001-E002) <a id="FIELD-NEURAL-RENDERING-001-E003"></a>[FIELD-NEURAL-RENDERING-001-E003](#FIELD-NEURAL-RENDERING-001-E003)

2022-2024 年，NeRF 和 neural graphics 开始成为可单独讨论的硬件负载。`RT-NeRF`、`Instant-NeRF`、`NeuRex`、`Hardware Acceleration of Neural Graphics`、A-SSCC/ISSCC/VLSI 的 neural-rendering silicon rows，把问题从传统 SLAM 前端扩展到 encoding、MLP、ray sampling、frame energy 和移动端 real-time rendering。<a id="FIELD-NEURAL-RENDERING-001-E005"></a>[FIELD-NEURAL-RENDERING-001-E005](#FIELD-NEURAL-RENDERING-001-E005) <a id="FIELD-NEURAL-RENDERING-001-E006"></a>[FIELD-NEURAL-RENDERING-001-E006](#FIELD-NEURAL-RENDERING-001-E006) <a id="FIELD-NEURAL-RENDERING-001-E007"></a>[FIELD-NEURAL-RENDERING-001-E007](#FIELD-NEURAL-RENDERING-001-E007) <a id="FIELD-NEURAL-RENDERING-001-E008"></a>[FIELD-NEURAL-RENDERING-001-E008](#FIELD-NEURAL-RENDERING-001-E008) <a id="FIELD-NEURAL-RENDERING-001-E009"></a>[FIELD-NEURAL-RENDERING-001-E009](#FIELD-NEURAL-RENDERING-001-E009) <a id="FIELD-NEURAL-RENDERING-001-E010"></a>[FIELD-NEURAL-RENDERING-001-E010](#FIELD-NEURAL-RENDERING-001-E010) <a id="FIELD-NEURAL-RENDERING-001-E011"></a>[FIELD-NEURAL-RENDERING-001-E011](#FIELD-NEURAL-RENDERING-001-E011)

2024 之后，3DGS 把瓶颈推到更显式的 rendering pipeline：projection、tile binning、sort length、alpha blending、Gaussian parameter movement、frame latency 和 quality-normalized throughput。cross-venue trend claim 把这一线描述为 emerging frontier，而不是已经完全稳定的 taxonomy boundary。<a id="FIELD-NEURAL-RENDERING-001-C003"></a>[FIELD-NEURAL-RENDERING-001-C003](#FIELD-NEURAL-RENDERING-001-C003)

## 核心问题

1. 排序和可见性：3DGS 不是 dense FLOP-heavy workload。GSCore、GS-TG、GSAcc、Neo 和 SwiftGS 都把 sort/reuse/visibility/blending 变成一等瓶颈。<a id="FIELD-NEURAL-RENDERING-001-E016"></a>[FIELD-NEURAL-RENDERING-001-E016](#FIELD-NEURAL-RENDERING-001-E016) <a id="FIELD-NEURAL-RENDERING-001-E023"></a>[FIELD-NEURAL-RENDERING-001-E023](#FIELD-NEURAL-RENDERING-001-E023) <a id="FIELD-NEURAL-RENDERING-001-E024"></a>[FIELD-NEURAL-RENDERING-001-E024](#FIELD-NEURAL-RENDERING-001-E024) <a id="FIELD-NEURAL-RENDERING-001-E037"></a>[FIELD-NEURAL-RENDERING-001-E037](#FIELD-NEURAL-RENDERING-001-E037) <a id="FIELD-NEURAL-RENDERING-001-E038"></a>[FIELD-NEURAL-RENDERING-001-E038](#FIELD-NEURAL-RENDERING-001-E038)

2. Neural field encoding 与 MLP pipeline：RT-NeRF、NeuRex、Booth-NeRF、CICC neural volume rendering、FlexNeRFer 等行支撑一个 NeRF/Instant-NGP 路线，核心矛盾是采样、encoding lookup、小 MLP 利用率和边缘设备帧延迟。[FIELD-NEURAL-RENDERING-001-E005](#FIELD-NEURAL-RENDERING-001-E005) [FIELD-NEURAL-RENDERING-001-E007](#FIELD-NEURAL-RENDERING-001-E007) <a id="FIELD-NEURAL-RENDERING-001-E012"></a>[FIELD-NEURAL-RENDERING-001-E012](#FIELD-NEURAL-RENDERING-001-E012) <a id="FIELD-NEURAL-RENDERING-001-E013"></a>[FIELD-NEURAL-RENDERING-001-E013](#FIELD-NEURAL-RENDERING-001-E013) <a id="FIELD-NEURAL-RENDERING-001-E032"></a>[FIELD-NEURAL-RENDERING-001-E032](#FIELD-NEURAL-RENDERING-001-E032)

3. Memory capacity 与 training/offload：Instant-NeRF、NeRPIM、GSArch、StreamingGS、CLM 和 GS-Scale 说明训练或大场景 3DGS 不能只看 rasterizer stage，GPU memory、host offload、near-memory/PIM 和 streaming traffic 也会决定系统边界。[FIELD-NEURAL-RENDERING-001-E006](#FIELD-NEURAL-RENDERING-001-E006) [FIELD-NEURAL-RENDERING-001-E011](#FIELD-NEURAL-RENDERING-001-E011) <a id="FIELD-NEURAL-RENDERING-001-E022"></a>[FIELD-NEURAL-RENDERING-001-E022](#FIELD-NEURAL-RENDERING-001-E022) <a id="FIELD-NEURAL-RENDERING-001-E025"></a>[FIELD-NEURAL-RENDERING-001-E025](#FIELD-NEURAL-RENDERING-001-E025) <a id="FIELD-NEURAL-RENDERING-001-E035"></a>[FIELD-NEURAL-RENDERING-001-E035](#FIELD-NEURAL-RENDERING-001-E035) <a id="FIELD-NEURAL-RENDERING-001-E036"></a>[FIELD-NEURAL-RENDERING-001-E036](#FIELD-NEURAL-RENDERING-001-E036)

4. Online SLAM / Gaussian-map update：GauSPU、RTGS、REACT3D、Splatonic、Space-Mate 和 NeRF-Learner 把 3DGS/NeRF 从离线 novel-view rendering 推向在线 mapping、tracking、training/inference 统一和移动端空间计算。<a id="FIELD-NEURAL-RENDERING-001-E018"></a>[FIELD-NEURAL-RENDERING-001-E018](#FIELD-NEURAL-RENDERING-001-E018) <a id="FIELD-NEURAL-RENDERING-001-E027"></a>[FIELD-NEURAL-RENDERING-001-E027](#FIELD-NEURAL-RENDERING-001-E027) <a id="FIELD-NEURAL-RENDERING-001-E028"></a>[FIELD-NEURAL-RENDERING-001-E028](#FIELD-NEURAL-RENDERING-001-E028) <a id="FIELD-NEURAL-RENDERING-001-E034"></a>[FIELD-NEURAL-RENDERING-001-E034](#FIELD-NEURAL-RENDERING-001-E034) <a id="FIELD-NEURAL-RENDERING-001-E039"></a>[FIELD-NEURAL-RENDERING-001-E039](#FIELD-NEURAL-RENDERING-001-E039) <a id="FIELD-NEURAL-RENDERING-001-E040"></a>[FIELD-NEURAL-RENDERING-001-E040](#FIELD-NEURAL-RENDERING-001-E040)

5. Evaluation 和 profiling：本领域需要记录 active Gaussian count、frame latency、rasterizer stage time、tile-list/sort length、render quality、optimizer/scene memory 和 tracking/update tail latency；只写 FPS 或 speedup 不足以比较路线。<a id="FIELD-NEURAL-RENDERING-001-C007"></a>[FIELD-NEURAL-RENDERING-001-C007](#FIELD-NEURAL-RENDERING-001-C007)

## 发展脉络

第一阶段是 SLAM/VIO substrate。FPL 2016、HOTCHIPS 2018 和 FPT 2022 的行不直接证明 NeRF/3DGS 成熟，但它们解释了为什么 scene-reconstruction pipeline 会进入硬件会议：相机输入、tracking、feature extraction、pose update 和嵌入式功耗约束已经是硬件问题。[FIELD-NEURAL-RENDERING-001-E001](#FIELD-NEURAL-RENDERING-001-E001) [FIELD-NEURAL-RENDERING-001-E002](#FIELD-NEURAL-RENDERING-001-E002) [FIELD-NEURAL-RENDERING-001-E003](#FIELD-NEURAL-RENDERING-001-E003)

第二阶段是 NeRF/neural graphics accelerator formation。ICCAD 2022 的 RT-NeRF、DAC 2023 的 Instant-NeRF、ISCA 2023 的 NeuRex 和 Hardware Acceleration of Neural Graphics，把 bottleneck 从传统视觉前端转到 encoding、sampling、MLP、near-memory 和 neural graphics execution。[FIELD-NEURAL-RENDERING-001-E005](#FIELD-NEURAL-RENDERING-001-E005) [FIELD-NEURAL-RENDERING-001-E006](#FIELD-NEURAL-RENDERING-001-E006) [FIELD-NEURAL-RENDERING-001-E007](#FIELD-NEURAL-RENDERING-001-E007) [FIELD-NEURAL-RENDERING-001-E008](#FIELD-NEURAL-RENDERING-001-E008)

第三阶段是 measured edge/silicon route。A-SSCC 2023、ISSCC 2023、VLSI 2023、CICC 2024 和 TCAD 2024 说明 neural rendering 不只在 architecture venue 里出现，也在 circuits/PIM/FPGA 语境下成为 frame-energy、PPA 和移动端 real-time 问题。[FIELD-NEURAL-RENDERING-001-E009](#FIELD-NEURAL-RENDERING-001-E009) [FIELD-NEURAL-RENDERING-001-E010](#FIELD-NEURAL-RENDERING-001-E010) [FIELD-NEURAL-RENDERING-001-E011](#FIELD-NEURAL-RENDERING-001-E011) [FIELD-NEURAL-RENDERING-001-E013](#FIELD-NEURAL-RENDERING-001-E013) <a id="FIELD-NEURAL-RENDERING-001-E014"></a>[FIELD-NEURAL-RENDERING-001-E014](#FIELD-NEURAL-RENDERING-001-E014)

第四阶段是 3DGS pipeline specialization。ASPLOS 2024 GSCore 是清晰的 3DGS architecture milestone candidate；A-SSCC 2024 Gaussian Splats processor、DATE 2025 FAMERS/PS-GS、HPCA/DAC/MICRO 2025 rows、ASPLOS 2026 Neo 和 MLSys 2026 SwiftGS 显示路线快速分化到 FPGA、edge GPU plug-in、sorting/rasterization、training memory 和 GPU system co-optimization。[FIELD-NEURAL-RENDERING-001-E016](#FIELD-NEURAL-RENDERING-001-E016) <a id="FIELD-NEURAL-RENDERING-001-E017"></a>[FIELD-NEURAL-RENDERING-001-E017](#FIELD-NEURAL-RENDERING-001-E017) <a id="FIELD-NEURAL-RENDERING-001-E019"></a>[FIELD-NEURAL-RENDERING-001-E019](#FIELD-NEURAL-RENDERING-001-E019) <a id="FIELD-NEURAL-RENDERING-001-E020"></a>[FIELD-NEURAL-RENDERING-001-E020](#FIELD-NEURAL-RENDERING-001-E020) <a id="FIELD-NEURAL-RENDERING-001-E021"></a>[FIELD-NEURAL-RENDERING-001-E021](#FIELD-NEURAL-RENDERING-001-E021) [FIELD-NEURAL-RENDERING-001-E022](#FIELD-NEURAL-RENDERING-001-E022) [FIELD-NEURAL-RENDERING-001-E023](#FIELD-NEURAL-RENDERING-001-E023) [FIELD-NEURAL-RENDERING-001-E024](#FIELD-NEURAL-RENDERING-001-E024) <a id="FIELD-NEURAL-RENDERING-001-E026"></a>[FIELD-NEURAL-RENDERING-001-E026](#FIELD-NEURAL-RENDERING-001-E026) [FIELD-NEURAL-RENDERING-001-E037](#FIELD-NEURAL-RENDERING-001-E037) [FIELD-NEURAL-RENDERING-001-E038](#FIELD-NEURAL-RENDERING-001-E038)

第五阶段是 online mapping 和 tool/benchmark formation。NeRArch-Sim 是 simulator/benchmark/DSE 信号，Splatonic、CLM、GS-Scale、Space-Mate、NeRF-Learner 等 2026 rows 则把 field frontier 推向 3DGS-SLAM、host offload、CIM 和 simultaneous rendering/tracking/training。<a id="FIELD-NEURAL-RENDERING-001-E033"></a>[FIELD-NEURAL-RENDERING-001-E033](#FIELD-NEURAL-RENDERING-001-E033) [FIELD-NEURAL-RENDERING-001-E034](#FIELD-NEURAL-RENDERING-001-E034) [FIELD-NEURAL-RENDERING-001-E035](#FIELD-NEURAL-RENDERING-001-E035) [FIELD-NEURAL-RENDERING-001-E036](#FIELD-NEURAL-RENDERING-001-E036) [FIELD-NEURAL-RENDERING-001-E039](#FIELD-NEURAL-RENDERING-001-E039) [FIELD-NEURAL-RENDERING-001-E040](#FIELD-NEURAL-RENDERING-001-E040)
