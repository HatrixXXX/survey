## 研究背景

[GSF-09](../40_synthesis/third_wave_field_synthesis.md#GSF-09) 从跨会议论文群里浮现，是因为多个 venue 反复遇到同一个工程矛盾：机器人和空间感知不是离线 batch workload，而是有 sensor deadline、tracking latency、map-update latency、power 和 accuracy/success-rate 共同约束的闭环计算问题。<a id="FIELD-EDGE-ROBOTICS-001-C011"></a>[FIELD-EDGE-ROBOTICS-001-C011](#FIELD-EDGE-ROBOTICS-001-C011)

早期证据集中在 sensor-near depth/stereo/optic-flow 和低功耗 3D vision。A-SSCC 2016 的 depth-estimation processor、ISSCC 2016 的 3D vision processor、VLSI 2017 的 nano-air-vehicle optic-flow image sensor，都把 perception front-end 当作能耗和时延问题处理。<a id="FIELD-EDGE-ROBOTICS-001-E001"></a>[FIELD-EDGE-ROBOTICS-001-E001](#FIELD-EDGE-ROBOTICS-001-E001) <a id="FIELD-EDGE-ROBOTICS-001-E002"></a>[FIELD-EDGE-ROBOTICS-001-E002](#FIELD-EDGE-ROBOTICS-001-E002) <a id="FIELD-EDGE-ROBOTICS-001-E005"></a>[FIELD-EDGE-ROBOTICS-001-E005](#FIELD-EDGE-ROBOTICS-001-E005)

随后，SLAM/VIO 进入 FPGA、SoC 和 silicon 语境。FPL 2016 `Semi-dense SLAM on an FPGA SoC`、HOTCHIPS 2018 `Navion...`、ISSCC/JSSC 的 CNN-SLAM、Navion article 和 NeuroSLAM rows，把问题从图像前处理推进到 tracking、pose update、VIO 和 integrated SLAM processor。<a id="FIELD-EDGE-ROBOTICS-001-E003"></a>[FIELD-EDGE-ROBOTICS-001-E003](#FIELD-EDGE-ROBOTICS-001-E003) <a id="FIELD-EDGE-ROBOTICS-001-E004"></a>[FIELD-EDGE-ROBOTICS-001-E004](#FIELD-EDGE-ROBOTICS-001-E004) <a id="FIELD-EDGE-ROBOTICS-001-E020"></a>[FIELD-EDGE-ROBOTICS-001-E020](#FIELD-EDGE-ROBOTICS-001-E020) <a id="FIELD-EDGE-ROBOTICS-001-E021"></a>[FIELD-EDGE-ROBOTICS-001-E021](#FIELD-EDGE-ROBOTICS-001-E021) <a id="FIELD-EDGE-ROBOTICS-001-E022"></a>[FIELD-EDGE-ROBOTICS-001-E022](#FIELD-EDGE-ROBOTICS-001-E022)

## 核心问题

1. Sensor-to-control latency：sensor-near depth/stereo/event/flow rows说明，前端数据搬运、预处理和压缩不能简单交给后端 accelerator 解决。[FIELD-EDGE-ROBOTICS-001-E001](#FIELD-EDGE-ROBOTICS-001-E001) [FIELD-EDGE-ROBOTICS-001-E005](#FIELD-EDGE-ROBOTICS-001-E005) <a id="FIELD-EDGE-ROBOTICS-001-E016"></a>[FIELD-EDGE-ROBOTICS-001-E016](#FIELD-EDGE-ROBOTICS-001-E016) <a id="FIELD-EDGE-ROBOTICS-001-E024"></a>[FIELD-EDGE-ROBOTICS-001-E024](#FIELD-EDGE-ROBOTICS-001-E024)

2. SLAM tracking and back-end update：feature extraction、keypoint selection、bundle adjustment、EKF update、mapping/navigation engine 和 direct tracking 形成一条 FPGA/SoC 路线。<a id="FIELD-EDGE-ROBOTICS-001-E006"></a>[FIELD-EDGE-ROBOTICS-001-E006](#FIELD-EDGE-ROBOTICS-001-E006) <a id="FIELD-EDGE-ROBOTICS-001-E014"></a>[FIELD-EDGE-ROBOTICS-001-E014](#FIELD-EDGE-ROBOTICS-001-E014) <a id="FIELD-EDGE-ROBOTICS-001-E015"></a>[FIELD-EDGE-ROBOTICS-001-E015](#FIELD-EDGE-ROBOTICS-001-E015) <a id="FIELD-EDGE-ROBOTICS-001-E017"></a>[FIELD-EDGE-ROBOTICS-001-E017](#FIELD-EDGE-ROBOTICS-001-E017) <a id="FIELD-EDGE-ROBOTICS-001-E018"></a>[FIELD-EDGE-ROBOTICS-001-E018](#FIELD-EDGE-ROBOTICS-001-E018) <a id="FIELD-EDGE-ROBOTICS-001-E019"></a>[FIELD-EDGE-ROBOTICS-001-E019](#FIELD-EDGE-ROBOTICS-001-E019)

3. Runtime adaptation and approximation：SLAMBooster、principled approximation、SlimSLAM 和 SuperNoVA 支撑一个动态适配路线，核心矛盾是 latency、energy 和 SLAM utility 之间的可控取舍。<a id="FIELD-EDGE-ROBOTICS-001-E008"></a>[FIELD-EDGE-ROBOTICS-001-E008](#FIELD-EDGE-ROBOTICS-001-E008) <a id="FIELD-EDGE-ROBOTICS-001-E009"></a>[FIELD-EDGE-ROBOTICS-001-E009](#FIELD-EDGE-ROBOTICS-001-E009) [FIELD-EDGE-ROBOTICS-001-E033](#FIELD-EDGE-ROBOTICS-001-E033) [FIELD-EDGE-ROBOTICS-001-E035](#FIELD-EDGE-ROBOTICS-001-E035)

4. Map, LiDAR and planning memory：VoxelCache、semantic LiDAR SLAM、GPPU 和 ray-casting accelerator 指向 map-state、point-cloud、path-planning 和 online 3D reconstruction 的 memory/locality 瓶颈。<a id="FIELD-EDGE-ROBOTICS-001-E010"></a>[FIELD-EDGE-ROBOTICS-001-E010](#FIELD-EDGE-ROBOTICS-001-E010) <a id="FIELD-EDGE-ROBOTICS-001-E023"></a>[FIELD-EDGE-ROBOTICS-001-E023](#FIELD-EDGE-ROBOTICS-001-E023) <a id="FIELD-EDGE-ROBOTICS-001-E025"></a>[FIELD-EDGE-ROBOTICS-001-E025](#FIELD-EDGE-ROBOTICS-001-E025) [FIELD-EDGE-ROBOTICS-001-E027](#FIELD-EDGE-ROBOTICS-001-E027)

5. Embodied policy and VLA latency：DATE 2026 robotic diffusion policy、VLSI 2026 VLA edge processor 和 SR-VLNA 是明确的 frontier signals，但不能单独升级成稳定边界；它们仍需要真实 robot trace、deadline 和 action-quality 证据。<a id="FIELD-EDGE-ROBOTICS-001-C023"></a>[FIELD-EDGE-ROBOTICS-001-C023](#FIELD-EDGE-ROBOTICS-001-C023) [FIELD-EDGE-ROBOTICS-001-E038](#FIELD-EDGE-ROBOTICS-001-E038) [FIELD-EDGE-ROBOTICS-001-E040](#FIELD-EDGE-ROBOTICS-001-E040) [FIELD-EDGE-ROBOTICS-001-E041](#FIELD-EDGE-ROBOTICS-001-E041)

## 研究方法

本报告把方法族分成六类：sensor-near depth/event vision、FPGA/SoC SLAM mapping、visual-inertial and mixed-signal SLAM silicon、runtime/approximation、map/LiDAR/planning memory、embodied-policy/VLA frontier。<a id="FIELD-EDGE-ROBOTICS-001-C021"></a>[FIELD-EDGE-ROBOTICS-001-C021](#FIELD-EDGE-ROBOTICS-001-C021)
