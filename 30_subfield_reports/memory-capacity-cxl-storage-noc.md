## 研究背景

[GSF-11](../40_synthesis/third_wave_field_synthesis.md#GSF-11) 从跨会议论文群中浮现，是因为不同社区反复遇到同一个结构性问题：compute kernel 变快后，容量、远端访问、host-device transfer、storage streaming、DMA fanout、HBM channel mapping、inter-die bandwidth 或 coherence semantics 开始限制系统结论。selected evidence matrix 选出 29 行，覆盖 persistent-memory workload、tiered/far memory、CXL/disaggregated memory、CXL bridge correctness、FPGA-HBM/NoC、storage/I/O-DMA、package/HBM/chiplet 和 memory evidence-validity [FIELD-MEMORY-CAPACITY-001-C005](#FIELD-MEMORY-CAPACITY-001-C005)。

third-wave textual clustering 对本领域有支持但不充分：SF04 提供 memory/cache/sparse vocabulary 支持，SF06 有较弱 embedded/management signal；SF11、SF14、SF98 只作为 boundary-audit residual，不能单独定义稳定小领域 <a id="FIELD-MEMORY-CAPACITY-001-C002"></a>[FIELD-MEMORY-CAPACITY-001-C002](#FIELD-MEMORY-CAPACITY-001-C002)。因此，本报告用 textual clustering 做漏检和边界复核，论文级事实仍回到 venue evidence、claim ledger、artifact audit 和 workload-artifact inventory。

workload-artifact inventory 还提示了 [GSF-11](../40_synthesis/third_wave_field_synthesis.md#GSF-11) 的评测字段：CXL/far-memory/tiered-memory 行需要记录 remote latency、bandwidth、page migration、synchronization、precision、energy/access 等边界；CXL specification 和 memory-pooling explainer 可以支撑术语和接口背景，但不是 runnable benchmark <a id="FIELD-MEMORY-CAPACITY-001-C006"></a>[FIELD-MEMORY-CAPACITY-001-C006](#FIELD-MEMORY-CAPACITY-001-C006)<a id="FIELD-MEMORY-CAPACITY-001-C007"></a>[FIELD-MEMORY-CAPACITY-001-C007](#FIELD-MEMORY-CAPACITY-001-C007)。
