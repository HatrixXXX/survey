## 研究背景

这个领域从跨 venue 证据中浮现，是因为加速器研究经常在更低层遇到同一类失败：架构或编译方案在抽象 simulator、C/HLS report 或 high-level cost model 中看起来成立，但实际芯片、FPGA 或 chiplet package 还要通过 timing、routability、power-grid、thermal、signal/power integrity、die-to-die link 和 package bandwidth 约束 <a id="FIELD-PHYSICAL-PPA-001-C003"></a>[FIELD-PHYSICAL-PPA-001-C003](#FIELD-PHYSICAL-PPA-001-C003)。

[GSF-05](../40_synthesis/third_wave_field_synthesis.md#GSF-05) 和 [GSF-04](../40_synthesis/third_wave_field_synthesis.md#GSF-04) HLS/compiler closure 相邻，但关注点更靠后端：[GSF-04](../40_synthesis/third_wave_field_synthesis.md#GSF-04) 关心从高层描述到硬件实现的生成闭环，[GSF-05](../40_synthesis/third_wave_field_synthesis.md#GSF-05) 关心生成后的物理实现、封装和硅后/封装级约束是否能闭合。它也和 [GSF-11](../40_synthesis/third_wave_field_synthesis.md#GSF-11) memory capacity/HBM/NoC 相邻，但 [GSF-05](../40_synthesis/third_wave_field_synthesis.md#GSF-05) 只在 HBM、UCIe、chiplet 和 3DIC 作为 package/physical closure bottleneck 时纳入，而不把所有 memory hierarchy 或 datacenter fabric 问题并入 <a id="FIELD-PHYSICAL-PPA-001-C012"></a>[FIELD-PHYSICAL-PPA-001-C012](#FIELD-PHYSICAL-PPA-001-C012)。

## 核心问题

1. Timing and routability predictability：placement、routing 和 pre-routing timing 需要在详细实现前就暴露风险，否则后续优化成本很高。DAC 2019 timing prediction、DATE 2022 FASTGR、TCAD 2023 DREAMPlace 4.0 和 ASP-DAC 2026 C3PO 都落在这条线上 <a id="FIELD-PHYSICAL-PPA-001-E006"></a>[FIELD-PHYSICAL-PPA-001-E006](#FIELD-PHYSICAL-PPA-001-E006)<a id="FIELD-PHYSICAL-PPA-001-E010"></a>[FIELD-PHYSICAL-PPA-001-E010](#FIELD-PHYSICAL-PPA-001-E010)<a id="FIELD-PHYSICAL-PPA-001-E019"></a>[FIELD-PHYSICAL-PPA-001-E019](#FIELD-PHYSICAL-PPA-001-E019)<a id="FIELD-PHYSICAL-PPA-001-E024"></a>[FIELD-PHYSICAL-PPA-001-E024](#FIELD-PHYSICAL-PPA-001-E024)。
2. Physical reliability and signoff：power-grid EM、IR-drop、PDN、capacitance extraction 和 thermal modeling 属于可实现性约束，不是性能评测的附属项。ICCAD 2016 EM checking、DATE 2016 capacitance extraction、PACT thermal simulator、DPO-3D 和 3D-ICE 4.0 支撑这条问题线 <a id="FIELD-PHYSICAL-PPA-001-E001"></a>[FIELD-PHYSICAL-PPA-001-E001](#FIELD-PHYSICAL-PPA-001-E001)<a id="FIELD-PHYSICAL-PPA-001-E002"></a>[FIELD-PHYSICAL-PPA-001-E002](#FIELD-PHYSICAL-PPA-001-E002)<a id="FIELD-PHYSICAL-PPA-001-E015"></a>[FIELD-PHYSICAL-PPA-001-E015](#FIELD-PHYSICAL-PPA-001-E015)<a id="FIELD-PHYSICAL-PPA-001-E025"></a>[FIELD-PHYSICAL-PPA-001-E025](#FIELD-PHYSICAL-PPA-001-E025)<a id="FIELD-PHYSICAL-PPA-001-E026"></a>[FIELD-PHYSICAL-PPA-001-E026](#FIELD-PHYSICAL-PPA-001-E026)。
3. FPGA implementation closure：FPGA 论文中的加速机制最终要过 congestion、routing runtime、memory footprint、timing 和 bitstream flow。FPL 2018 congestion estimation、Yosys+nextpnr、FCCM 2025 routing convergence 和 FPL 2025 Routino 是代表行 <a id="FIELD-PHYSICAL-PPA-001-E005"></a>[FIELD-PHYSICAL-PPA-001-E005](#FIELD-PHYSICAL-PPA-001-E005)<a id="FIELD-PHYSICAL-PPA-001-E007"></a>[FIELD-PHYSICAL-PPA-001-E007](#FIELD-PHYSICAL-PPA-001-E007)<a id="FIELD-PHYSICAL-PPA-001-E022"></a>[FIELD-PHYSICAL-PPA-001-E022](#FIELD-PHYSICAL-PPA-001-E022)<a id="FIELD-PHYSICAL-PPA-001-E023"></a>[FIELD-PHYSICAL-PPA-001-E023](#FIELD-PHYSICAL-PPA-001-E023)。

Evidence validity：很多 source 是官方 program、DBLP、abstract、slides 或 vendor presentation，能支撑阅读路线，但不能直接证明 adoption、local-execution validity 或标准化 [FIELD-PHYSICAL-PPA-001-C005](#FIELD-PHYSICAL-PPA-001-C005)<a id="FIELD-PHYSICAL-PPA-001-C009"></a>[FIELD-PHYSICAL-PPA-001-C009](#FIELD-PHYSICAL-PPA-001-C009)。

## 研究方法

- Physical signoff models：EM checking、capacitance extraction、PDN/IR-drop optimization、thermal simulation and modeling。这些方法说明 physical closure 是独立证据层，不是论文最后补一张 PPA 表 <a id="FIELD-PHYSICAL-PPA-001-C006"></a>[FIELD-PHYSICAL-PPA-001-C006](#FIELD-PHYSICAL-PPA-001-C006)[FIELD-PHYSICAL-PPA-001-C009](#FIELD-PHYSICAL-PPA-001-C009)。
- Predictive and accelerated CAD：ML congestion estimation、pre-routing timing prediction、CPU-GPU global routing、timing-driven placement、commercial-quality global placement。这条线的共同目标是减少 late-stage closure failure 和昂贵的迭代 [FIELD-PHYSICAL-PPA-001-C006](#FIELD-PHYSICAL-PPA-001-C006)。
- FPGA routing and bitstream flow：Yosys+nextpnr、PathFinder convergence analysis 和 routing-memory representation 说明 FPGA closure 需要真实工具链、routing convergence、runtime 和 memory evidence <a id="FIELD-PHYSICAL-PPA-001-C008"></a>[FIELD-PHYSICAL-PPA-001-C008](#FIELD-PHYSICAL-PPA-001-C008)。
- Chiplet/package modeling：chiplet cost modeling、HBM package integration、inter-chiplet SI/PI、UCIe usage/protocol、die-to-die transceiver、3D FPGA CAD、3DIC PDN 和 thermal model 是 package closure 的方法族 <a id="FIELD-PHYSICAL-PPA-001-C007"></a>[FIELD-PHYSICAL-PPA-001-C007](#FIELD-PHYSICAL-PPA-001-C007)。
- Evaluation and artifact discipline：OpenMPL、PACT、Yosys+nextpnr、3D reconfigurable tools 和 3D-ICE 是 tool/benchmark-style evidence，但没有本任务本地执行，因此只能写 availability/signal，不写 locally executed [FIELD-PHYSICAL-PPA-001-C005](#FIELD-PHYSICAL-PPA-001-C005)[FIELD-PHYSICAL-PPA-001-C009](#FIELD-PHYSICAL-PPA-001-C009)。

## 开放问题

1. workload绑定仍然很弱。许多 EDA/package 行是基础设施或设计方法论文，而不是特定于workload的加速器证据，与 [GSF-05](../40_synthesis/third_wave_field_synthesis.md#GSF-05) [FIELD-PHYSICAL-PPA-001-C012](#FIELD-PHYSICAL-PPA-001-C012) 的库存警告相匹配。

## 连接到用户方向

对于 FPGA 研究，[GSF-05](../40_synthesis/third_wave_field_synthesis.md#GSF-05) 是 HLS 和加速器映射下方的物理闭合层。它告诉用户检查拥塞、路由收敛、时序松弛、比特流流量、路由内存/runtime和物理感知布局规划，而不是停留在kernel II 或模拟加速 [FIELD-PHYSICAL-PPA-001-C008](#FIELD-PHYSICAL-PPA-001-C008) 处。

对于 3DGS/神经渲染加速，该场是相邻的而不是中心的。当渲染器、光栅器、排序器、混合管道或内存子系统映射到 FPGA/ASIC/chiplet 硬件并且必须关闭时序、路由、封装带宽或热限制时，这一点很重要。它不会使 3D 场景重建成为物理/PPA 闭包 <a id="FIELD-PHYSICAL-PPA-001-C004"></a>[FIELD-PHYSICAL-PPA-001-C004](#FIELD-PHYSICAL-PPA-001-C004) 的一部分。

对于机器人和嵌入式边缘计算，当边缘加速器从模型/runtimeclaim转向硅或封装约束时，[GSF-05](../40_synthesis/third_wave_field_synthesis.md#GSF-05) 变得相关：热包络、封装带宽、芯片到芯片链路行为、HBM 可用性和板/芯片功率对可部署性很重要，但它们应该被测量或获取，而不是推断 [FIELD-PHYSICAL-PPA-001-C005](#FIELD-PHYSICAL-PPA-001-C005)[FIELD-PHYSICAL-PPA-001-C007](#FIELD-PHYSICAL-PPA-001-C007)。
